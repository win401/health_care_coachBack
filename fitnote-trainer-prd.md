# FitNote Trainer PRD

## 1. 개요

### 제품명

FitNote Trainer

### 한 줄 설명

헬스트레이너가 회원별 수업 기록, 과제, 주의사항을 관리하고, 회원 사진 기반 자세 분석 결과를 활용해 더 체계적인 피드백을 제공할 수 있도록 돕는 PT 회원관리·자세분석 도우미 서비스

### 만드는 이유

헬스트레이너는 여러 회원의 운동 목표, 부상 이력, 수업 내용, 과제 수행 여부를 동시에 관리해야 한다. 하지만 실제 현장에서는 카카오톡, 개인 메모, 기억에 의존하는 경우가 많아 수업 기록 누락, 회원 상태 혼동, 컴플레인 대응 어려움이 발생한다.

또한 회원은 PT 수업을 받고 있어도 자신의 자세가 얼마나 개선되고 있는지 눈으로 확인하기 어렵다. 트레이너가 말로 설명해도 회원 입장에서는 “내 자세가 정확히 어디가 문제인지”, “정석 자세와 얼마나 차이가 나는지”를 직관적으로 이해하기 어렵다.

FitNote Trainer는 트레이너가 회원 정보를 체계적으로 관리하고, 회원 사진을 기반으로 자세 분석 결과를 제공해 현재 자세와 기준 자세의 차이를 시각적으로 보여준다. 이를 통해 트레이너는 설명력 있는 피드백을 제공하고, 회원은 자신이 관리받고 있다는 신뢰와 개선 방향을 확인할 수 있다.

### 프로젝트 가이드 기준

- 프로젝트 주제: 헬스케어 AI 기반 건강관리 서비스
- 총 기간: 4일
- 일일 진행 시간: 4시간
- 총 학습/개발 시간: 16시간
- 개발 환경: Google Colab, Gradio, SQLite, Python
- 주요 산출물: 페인포인트, 저니맵, PRD, Process Flow, ERD, Colab 노트북, 발표 자료

## 2. 목표

### 비즈니스 목표

- 트레이너의 회원관리 업무 부담을 줄인다.
- PT 수업 기록 누락으로 인한 회원 컴플레인을 줄인다.
- 자세 분석 결과를 통해 트레이너의 피드백 신뢰도를 높인다.
- 회원이 자신의 변화 과정을 눈으로 확인하게 하여 재등록 가능성을 높인다.

### 사용자 목표

- 트레이너는 회원별 목표, 주의사항, 이전 수업 내용을 빠르게 확인할 수 있다.
- 트레이너는 수업 후 세션 내용을 쉽게 기록하고 요약할 수 있다.
- 트레이너는 회원 사진 기반 자세 분석 결과를 참고해 더 구체적인 피드백을 제공할 수 있다.
- 회원은 현재 자세와 정석 자세의 차이를 시각적으로 확인할 수 있다.
- 회원은 오늘 수업 내용, 자세 피드백, 다음 과제를 명확히 확인할 수 있다.

### 성공 지표, KPI

- 트레이너가 회원 1명을 1분 이내에 등록할 수 있다.
- 트레이너가 수업 후 세션 기록을 2분 이내에 작성할 수 있다.
- 회원별 이전 수업 기록과 주의사항을 한 화면에서 확인할 수 있다.
- 회원 사진 업로드 후 자세 분석 결과 화면을 확인할 수 있다.
- 발표 데모에서 `회원 선택 -> 사진 업로드 -> 자세 분석 결과 확인 -> 세션 기록 -> 과제 등록` 흐름을 완성한다.
- 팀 내부 테스트 기준 만족도 4.0/5.0 이상을 목표로 한다.

## 4. 핵심 기능

### Must Have

#### F1. 회원 프로필 등록

트레이너는 회원의 기본 정보, 운동 목표, 운동 경험, 부상·통증 주의사항을 등록할 수 있다.

- 입력 항목: 이름, 나이, 성별, 운동 목표, 운동 경험, 통증 부위, 주의사항
- 목적: 수업 전 회원 상태를 빠르게 파악하기 위함

#### F2. 회원별 세션 기록 작성

트레이너는 PT 수업 후 해당 회원의 세션 내용을 기록할 수 있다.

- 입력 항목: 운동 부위, 운동 종목, 세트/횟수, 오늘의 특이사항, 다음 수업 메모
- 목적: 이전 수업과 다음 수업을 자연스럽게 연결하기 위함

#### F3. 회원 사진 업로드

트레이너 또는 회원은 자세 확인을 위해 운동 자세 사진을 업로드할 수 있다.

- 입력 항목: 회원 선택, 운동 동작명, 자세 사진
- 예시 동작: 스쿼트, 런지, 플랭크, 데드리프트 준비 자세
- 목적: 현재 자세를 기록하고 분석하기 위함

#### F4. YOLO 기반 자세 분석

시스템은 업로드된 회원 사진을 기반으로 주요 신체 포인트를 탐지하고, 현재 자세를 분석한다.

- 분석 방식: YOLO 또는 포즈 추정 모델을 활용해 어깨, 골반, 무릎, 발목 등 주요 포인트를 인식
- 출력 예시: 관절 위치, 신체 정렬, 무릎 방향, 상체 기울기, 좌우 균형
- 목적: 트레이너가 자세 피드백을 시각적 근거와 함께 제공하도록 돕기 위함

#### F5. 정석 자세와 현재 자세 비교

시스템은 기준 자세와 현재 회원 자세의 차이를 점수와 시각 자료로 보여준다.

- 출력 항목: 자세 점수, 주요 차이점, 개선 필요 부위, 시각화 이미지
- 예시: “무릎이 안쪽으로 모이는 경향”, “상체가 기준보다 앞으로 기울어짐”
- 목적: 회원이 자신의 자세 문제를 눈으로 이해하도록 돕기 위함

#### F6. 트레이너 피드백 작성

트레이너는 자세 분석 결과를 참고해 회원에게 전달할 피드백을 작성할 수 있다.

- 입력 항목: 트레이너 코멘트, 주의사항, 다음 수업 확인사항
- 목적: AI 분석 결과를 트레이너 판단과 결합하기 위함
- 원칙: 시스템 분석 결과는 참고 자료이며, 최종 피드백은 트레이너가 작성한다.

#### F7. 음성 기록 기반 세션 요약

트레이너는 수업 후 음성으로 세션 내용을 남기고, 시스템은 이를 요약 정리 폼에 맞게 변환한다.

- 실제 목표 기능: 음성녹음 -> 백엔드 전송 -> STT 변환 -> 키워드 추출 -> 요약 폼 자동 채움
- 발표 데모 범위: 실제 음성녹음/STT는 구현하지 않고, 텍스트화된 예시 세션 데이터를 입력해 요약 폼 자동 채움처럼 시연
- 목적: 트레이너의 기록 입력 부담을 줄이기 위함

#### F8. 과제 등록

트레이너는 회원에게 다음 수업 전까지 수행할 운동 과제를 등록할 수 있다.

- 예시: 벽 스쿼트 20회, 고관절 스트레칭 10분, 폼롤러 스트레칭 10분
- 목적: 수업 외 시간에도 관리받는 느낌을 제공하기 위함

#### F9. 회원별 요약 카드

트레이너는 다음 수업 전 회원의 목표, 주의사항, 최근 수업 내용, 자세 분석 결과, 진행 중 과제를 한눈에 확인할 수 있다.

- 목적: 회원별 상황을 빠르게 떠올리고 수업 연속성을 높이기 위함

#### F10. 간단 리포트 생성

시스템은 세션 기록, 자세 분석 결과, 과제를 기반으로 회원에게 보여줄 수 있는 간단 요약 리포트를 생성한다.

- 포함 내용: 오늘의 운동 요약, 자세 분석 결과, 잘한 점, 주의할 점, 다음 과제
- 목적: 회원에게 체계적으로 관리받고 있다는 인상을 주기 위함

## 5. 사용자 스토리

### US-01. 회원 등록

트레이너로서, 신규 회원의 목표와 주의사항을 빠르게 등록하고 싶다. 그래야 다음 수업에서 회원 상태를 잊지 않고 맞춤형 수업을 준비할 수 있다.

인수 조건:

- 회원 이름, 운동 목표, 주의사항을 입력할 수 있다.
- 등록된 회원은 목록에서 확인할 수 있다.
- 회원을 선택하면 상세 프로필을 볼 수 있다.

### US-02. 세션 기록

트레이너로서, PT 수업 후 오늘 진행한 운동과 특이사항을 기록하고 싶다. 그래야 다음 수업 때 이전 내용을 이어서 관리할 수 있다.

인수 조건:

- 회원별로 세션 기록을 작성할 수 있다.
- 운동 부위, 운동 내용, 세트/횟수, 메모를 입력할 수 있다.
- 저장된 세션 기록은 회원 상세 화면에서 확인할 수 있다.

### US-03. 사진 기반 자세 분석

트레이너로서, 회원의 운동 자세 사진을 업로드해 자세 분석 결과를 확인하고 싶다. 그래야 말로만 설명하지 않고 시각 자료를 바탕으로 피드백할 수 있다.

인수 조건:

- 회원을 선택한 뒤 운동 자세 사진을 업로드할 수 있다.
- 시스템은 사진에서 주요 신체 포인트를 탐지한다.
- 현재 자세의 문제 부위와 자세 점수를 표시한다.
- 분석 결과는 트레이너가 수정하거나 코멘트를 추가할 수 있다.

### US-04. 정석 자세와 비교

회원으로서, 내 현재 자세가 정석 자세와 어떻게 다른지 눈으로 확인하고 싶다. 그래야 내가 무엇을 고쳐야 하는지 직관적으로 이해할 수 있다.

인수 조건:

- 현재 자세 이미지와 기준 자세 차이를 시각화한다.
- 자세 점수 또는 개선 필요 항목을 보여준다.
- 분석 결과는 의료 진단이 아니라 운동 자세 참고 자료로 표시한다.

### US-05. 음성 기반 요약

트레이너로서, 수업 후 긴 글을 직접 쓰지 않고 음성이나 짧은 메모로 세션 요약을 만들고 싶다. 그래야 바쁜 수업 사이에도 기록을 남길 수 있다.

인수 조건:

- 데모에서는 텍스트화된 예시 세션을 입력한다.
- 시스템은 입력 텍스트에서 운동 내용, 주의사항, 과제 후보를 추출한다.
- 추출 결과는 요약 폼에 자동으로 채워진다.
- 최종 저장 전 트레이너가 수정할 수 있다.

### US-06. 과제 등록

트레이너로서, 회원에게 다음 수업 전까지 해야 할 과제를 등록하고 싶다. 그래야 수업 외 시간에도 운동 습관을 이어갈 수 있다.

인수 조건:

- 과제명, 내용, 목표량, 마감일을 입력할 수 있다.
- 과제는 회원 상세 화면에 표시된다.
- 과제 상태는 미확인, 진행 중, 완료로 구분할 수 있다.

### US-07. 회원 요약 확인

트레이너로서, 다음 수업 전에 회원의 목표, 주의사항, 최근 기록, 자세 분석 결과를 한눈에 보고 싶다. 그래야 회원에게 “나를 기억하고 있다”는 신뢰를 줄 수 있다.

인수 조건:

- 회원 상세 화면에서 목표, 주의사항, 최근 세션, 자세 분석 결과, 진행 중 과제를 볼 수 있다.
- 부상·통증 주의사항은 눈에 띄게 표시된다.
- 최근 자세 분석 점수와 주요 개선 포인트가 표시된다.

### US-08. 회원 리포트 확인

회원으로서, 오늘 내가 어떤 운동을 했고 내 자세에서 무엇을 개선해야 하는지 알고 싶다. 그래야 PT 수업 이후에도 운동 방향을 잊지 않을 수 있다.

인수 조건:

- 오늘의 운동 요약을 확인할 수 있다.
- 자세 분석 결과와 트레이너 피드백을 확인할 수 있다.
- 다음 과제를 확인할 수 있다.

### 주요 기술 구성

- 회원 데이터 저장
- 세션 기록 저장
- 과제 데이터 저장
- 이미지 업로드 기능
- YOLO 기반 자세 분석 또는 포즈 추정
- 자세 점수 계산
- 분석 결과 이미지 시각화
- 텍스트 기반 세션 요약 처리
- 키워드 추출 또는 규칙 기반 분류
- Gradio 기반 데모 화면

### YOLO 자세 분석 처리 방식

사용자가 운동 자세 사진을 업로드하면 시스템은 YOLO 기반 모델을 이용해 사람의 주요 신체 포인트를 탐지한다. 이후 기준 자세와 비교해 관절 각도, 좌우 균형, 무릎 방향, 상체 기울기 등의 차이를 계산하고, 이를 점수와 시각 자료로 제공한다.

단, 4일 미니 프로젝트에서는 모든 운동 동작을 분석하지 않고, 스쿼트 자세 1개 동작을 우선 분석 대상으로 제한한다.

### 자세 점수 계산 예시

| 항목 | 점수 |
| --- | ---: |
| 무릎 정렬 | 30점 |
| 상체 기울기 | 25점 |
| 골반 균형 | 25점 |
| 발 위치/하체 안정성 | 20점 |
| 총점 | 100점 |

### 예시 자세 분석 출력

- 자세 점수: 72점
- 주요 문제: 무릎이 안쪽으로 모이는 경향
- 보조 문제: 상체가 기준보다 앞으로 기울어짐
- 트레이너 피드백: “무릎이 발끝 방향과 같은 방향을 향하도록 벽 스쿼트 연습을 진행하세요.”
- 추천 과제: 벽 스쿼트 20회, 고관절 스트레칭 10분

### 음성/STT 기능 처리 방식

실제 서비스에서는 음성녹음 파일을 백엔드에서 받아 STT를 통해 텍스트로 변환하고, AI가 세션 요약 폼에 맞게 정리하는 구조를 목표로 한다.

다만 4일 미니 프로젝트에서는 실제 음성녹음과 STT API 연동은 구현하지 않고, “음성녹음이 텍스트로 변환되었다고 가정한 예시 세션 텍스트”를 입력값으로 사용한다.

예시 세션 텍스트 입력:

> 오늘 박수진 회원은 하체 운동을 진행했다. 스쿼트 3세트 12회, 런지 2세트 10회를 수행했다. 무릎이 안쪽으로 모이는 경향이 있어 다음 수업 때 자세 확인이 필요하다. 과제로는 벽 스쿼트 20회와 고관절 스트레칭 10분을 안내했다.

예시 세션 요약 출력:

- 운동 부위: 하체
- 수행 운동: 스쿼트 3세트 12회, 런지 2세트 10회
- 주의사항: 무릎이 안쪽으로 모이는 경향
- 다음 확인사항: 스쿼트 자세 확인
- 과제: 벽 스쿼트 20회, 고관절 스트레칭 10분

## 7. 제약사항

### 일정 제약

- 총 프로젝트 기간은 4일이다.
- 하루 작업 가능 시간은 약 4시간이며, 총 16시간 내 결과물을 만들어야 한다.
- 발표자료 제작과 발표 준비 시간이 포함되어야 한다.

### 범위 제약

- 실제 상용 서비스 수준의 회원관리 시스템을 만들지 않는다.
- MVP 데모는 핵심 흐름인 회원 등록, 사진 업로드, 자세 분석, 세션 기록, 과제 등록에 집중한다.
- 자세 분석은 우선 스쿼트 1개 동작으로 제한한다.
- 결제, 출석 관리, 센터 ERP, 식단 자동 분석, 완전 자동 운동 처방은 제외한다.

### 기술 제약

- 실제 음성녹음, STT API, AI API 연동은 구현하지 않는다.
- 음성 전사 기능은 텍스트화된 샘플 데이터를 통해 데모한다.
- YOLO 자세 분석은 데모 가능한 수준으로 구현하며, 실제 전문 트레이너 수준의 정확한 자세 진단을 목표로 하지 않는다.
- 기준 자세와의 비교 점수는 발표용 규칙 기반 점수로 사용한다.
- 데이터는 SQLite, CSV, JSON 등 간단한 방식으로 저장한다.

### 데이터 제약

- 실제 회원 개인정보를 사용하지 않고 가상 데이터를 사용한다.
- 자세 사진은 데모용 샘플 이미지를 사용한다.
- 부상, 통증 등 민감할 수 있는 정보는 데모 데이터로만 사용한다.
- 의료 진단이나 치료 조언으로 오해될 수 있는 표현은 피한다.

### 팀 제약

- 팀원은 3명이다.
- 역할은 PM/기획, UI/프로토타입, 데이터/로직 구현으로 나눈다.
- 구현이 어려운 기능은 발표용 시나리오와 더미 데이터로 대체한다.
